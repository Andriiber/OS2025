**Команда tar (tape archive)**

Призначення:
Створює архів із кількох файлів або папок (без стискання або з ним).

Основні параметри:

      -c — створити архів
      -x — розпакувати архів
      -v — показувати процес (verbose)
      -f — вказати ім’я файлу архіву
      -z — використовувати gzip для стиснення
      -j — використовувати bzip2 для стиснення
      -J — використовувати xz для стиснення

Встановлення:

```bash
sudo apt install tar
```

**Команда gzip**

**Призначення:**
Стискає окремі файли у формат .gz

Основні параметри:
      
      -c — вивести результат у стандартний потік
      -d — розпакувати файл
      -v — показати інформацію про стискання
      -k — зберегти оригінальний файл

Встановлення:

```bash
sudo apt install gzip
```

**Команда bzip2**

Призначення:
Стискає файли у формат .bz2 (ефективніше за gzip, але повільніше).

Основні параметри:

      -d — розпакувати
      -k — зберегти оригінал
      -v — показати інформацію
      -z — стискати (за замовчуванням)

Встановлення:

```bash
sudo apt install bzip2
```

**Команда xz**

Призначення:
Стискає файли у формат .xz (ще краще стискання, ніж bzip2).

Основні параметри:

      -d — розпакувати
      -k — зберегти оригінал
      -v — показати інформацію
      -z — стискати

Встановлення:

```bash
sudo apt install xz-utils
```

**Команда zip**

Призначення:
Створює архів у форматі .zip (популярний у Windows).

Основні параметри:

      -r — рекурсивно додати папки
      -v — показати процес
      -q — тихий режим
      -u — оновити існуючий архів

Встановлення:

```bash
sudo apt install zip unzip
```

| Команда  | Що виконує команда? |
| ----- | ----- |
| cmd 1\> file | Перенаправляє stdout (1) команди cmd у файл file, stderr залишається в консолі. |
| cmd \> file | Те саме, що cmd 1> file. Стандартний вихід йде у файл, стандартний потік помилок у консоль. |
| cmd 2\> file | Перенаправляє stderr (2) у файл file, stdout залишається у консолі. |
| cmd \>\> file | Додає stdout до файлу file (не перезаписує). stderr залишається у консолі. |
| cmd &\> file | Перенаправляє одночасно stdout і stderr у файл file. (Багато оболонок підтримують цей синтаксис). |
| cmd \> file 2\>&1 | Перенаправляє stdout у файл, а потім stderr у той же потік, куди пішов stdout. Ефект: обидва потоки в file. |
| cmd \>\> file 2\>&1 | Те саме, що попереднє, але stdout додається до файлу, а stderr йде туди ж. |
| cmd 2\>&1 \> /dev/null | Спочатку stderr підключається до stdout, потім stdout перенаправляється в /dev/null. Ефект: stdout в /dev/null, а stderr йде в консоль (бо stderr було підключено до старого stdout, який ще не перенаправлений). |
| cmd 2\> /dev/null | Відкидає stderr, stdout залишається в консолі. |
| cmd1 | Виконує cmd1, stdout і stderr виводяться в консоль. |
|cmd2 | Виконує cmd2, stdout і stderr в консолі. |
| cmd1 2\>&1 | Перенаправляє stderr у stdout, але оскільки stdout не перенаправлений, обидва потоки йдуть у консоль. |
| cmd2 | Виконує cmd2, стандартний вивід і помилки в консолі. |

| $ find \-name '\*.txt' \> file.txt 2\> /dev/null | Шукає всі файли з розширенням .txt, виводить їх список у file.txt, а повідомлення про помилки відправляє в /dev/null (ігнорує). | stdout → file.txt, stderr → /dev/null |
| $ cat file1\_unsorted / sort \> file2\_sorted | Зчитує вміст file1_unsorted, передає його команді sort для сортування, а результат записує у file2_sorted. | stdout (cat) → stdin (sort)` |
| $ cat myfile / grep student / wc \-l | Зчитує вміст файлу myfile, знаходить усі рядки, які містять слово "student", і підраховує, скільки таких рядків у файлі. | stdout (cat) → stdin (grep) → stdin (wc -l) → stdout (термінал) |

7. \*\*Яке призначення директорії файлу /dev/null?

         /dev/null — спеціальний "файл-поглинач".

Призначення:
      
      Всі дані, що записуються в /dev/null, губляться без можливості відновлення.
      
      Використовується для ігнорування виводу команд або помилок, наприклад: command > /dev/null 2>&1
