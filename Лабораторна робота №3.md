5. Дайте відповіді на наступні питання:

- Яку базову інформацію надає рядок запрошення prompt?
    
    Рядок запрошення (prompt) — це текст у терміналі, який сигналізує користувачу про готовність оболонки приймати команди. Зазвичай він містить:
    
        * Ім’я користувача (user)
        * Ім’я комп’ютера або хосту (host)
        * Поточний каталог (current directory)
        * Символ, що позначає завершення (наприклад, $ для звичайного користувача, # для root)

- Для чого команді потрібні параметри та аргументи?
    
        * Аргументи — це об’єкти, над якими виконується команда (наприклад, імена файлів чи каталогів).
        * Параметри (опції) — це ключі або прапорці, які змінюють поведінку команди (наприклад, -l для детального виведення).

    Приклад:
    
        bash
        ls -l /home/user/Documents
    
    
    Тут /home/user/Documents — аргумент, -l — параметр.

- Яке призначення команд ls, які параметри та аргументи вона може мати? Наведіть 3 приклади.

    Команда ls використовується для відображення списку файлів і каталогів у поточному каталозі або вказаному шляху.
    
    Параметри та аргументи:
    
         -l — детальна інформація (права доступу, власник, розмір)
         -a — показати всі файли, включно з прихованими
         -h — виводити розмір файлів у зручному форматі (KB, MB)
    
    Приклади:
    
        bash
        ls -l
        ls -a /home/user
        ls -lh /var/log


- Яким чином можна використати історію команд, які переваги це надає?
    
    Команда history або клавіші ↑/↓ дозволяють переглядати попередні команди.
    Переваги:
        
        Швидке повторення або редагування команд
        Легке виправлення помилок
        Автоматизація повторюваних завдань
    
    Приклад:
    
        bash
        history
        !42   # виконує команду під номером 42



- Яке призначення команди echo?

Команда echo використовується для виведення тексту або значення змінних у термінал.

Приклад:

    bash
    echo "Hello, world!"
    echo $HOME


- Охарактеризуйте поняття змінної в оболонці Bash, які типи змінних вона підтримує? 

    Змінна — це іменований контейнер для збереження даних (текст, числа, шляхи).
    
    Типи змінних:
    
        Локальні змінні** — існують тільки в поточній сесії оболонки
        Глобальні (експортовані) змінні** — доступні для дочірніх процесів
        Спеціальні змінні** — наприклад, `$HOME`, `$PATH`, `$?`
    
    Приклад створення змінної:
    
        bash
        NAME="Ivan"
        echo $NAME

- Яке призначення команд env, export та unset?

            env — показує всі поточні змінні середовища
            export — робить локальну змінну глобальною, доступною для дочірніх процесів
            unset — видаляє змінну
    
    Приклад:
    
        bash
        MYVAR="test"
        export MYVAR
        unset MYVAR

- Які команди для отримання довідки по командам в терміналі ви знаєте?

        man <команда> — відображає повну документацію
        <команда> --help — коротка довідка
        info <команда> — розширена документація (не завжди доступна)
        whatis <команда> — короткий опис

Приклад:
    
    bash
    man ls
    ls --help

1. Які типи команд існують в оболонці Bash?

    В Bash команди можна розділити на кілька типів:
    
    1. Вбудовані команди (builtin) – команди, реалізовані всередині Bash, наприклад:
    
           cd, echo, pwd, export, history, alias.ss
    
    2. Зовнішні команди (external) – це програми чи скрипти, які знаходяться у файловій системі, наприклад:
    
           /bin/ls, /usr/bin/python, /usr/bin/grep.
    
    4. Функції Bash – користувацькі або системні функції, визначені всередині Bash.
       Приклад:
    
           bash
           myfunc() { echo "Hello"; }
           myfunc
       
    
    5. Аліаси (aliases) – скорочення для команд, наприклад:
    
           bash
           alias ll='ls -l'
  

2. Що таке змінні оточення? Які вони бувають. Як їх можна переглянути в терміналі?

    Змінні оточення (environment variables) – це змінні, які доступні всім процесам, запущеним від поточного сеансу Bash.
    
    Типи змінних оточення:
    
    1. Стандартні змінні – надають інформацію про середовище:
    
           HOME – домашній каталог користувача
           PATH – шляхи для пошуку виконуваних файлів
           USER – ім’я користувача
           SHELL – оболонка користувача
    
    2. Користувацькі змінні – створюються самим користувачем:
    
           bash
           export MY_VAR="Hello"
           
    
    Переглянути змінні оточення можна так:
    
            env – показує всі змінні оточення
            printenv – аналогічно
            set – показує всі змінні, включно з локальними


3. *Опишіть змінну $PS1. Як в терміналі переглянути її вміст?

    $PS1 – це змінна, яка відповідає за рядок запрошення у Bash. 
    
    Наприклад, стандартний вигляд:
    
          user@hostname:~$
      
    Переглянути її можна так:
    
          bash
          echo $PS1
  

4. *Як можна змінити значення змінної $PS1? Що при цьому відбудеться в рядку запрошенні в bash (рядок запрошення перед початком кожної команди). Як змінити значення цієї змінної не на поточний сеанс, а за замовчуванням?
    
    Змінити значення можна так:
    
        bash
        PS1=">>> "
    
    
    Після цього рядок запрошення зміниться на >>> .
    
    Щоб зміни були постійними, потрібно додати цю команду в один із файлів конфігурації Bash, наприклад:
    
        ~/.bashrc – для інтерактивних сеансів
        ~/.bash_profile – для сеансів логіну
    
    Наприклад:
    
        bash
        echo 'PS1=">>> "' >> ~/.bashrc
        source ~/.bashrc
