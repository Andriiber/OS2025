2.1. *Які команди для моніторингу стану процесів ви знаєте. Як переглянути їх можливі параметри?

Основні команди для моніторингу процесів у Linux/Unix:

| Команда    | Призначення                                             
| ---------|-------------------------------------------------------
| ps| Виводить список поточних процесів                       
| ps axZ | Щоб отримати інформацію про безпеку:                        
| ps -U root -u root u | Щоб побачити кожен процес, що працює від імені root (real & effective ID), у форматі користувача  
| ps -ejh  | Відображає процеси у вигляді дерева                                     
| ps ax  | Щоб побачити кожен процес у системі за допомогою синтаксису BSD                

2.2. *Чи може команда ps у реальному часі відслідковувати стан процесів?

Ні, Команда ps показує знімок (snapshot) стану процесів на момент виконання команди.
Вона не оновлюється автоматично в реальному часі.

2.3. **За якими параметрами можливе сортування процесів в команді top? Як переключатись між ними?

У top можна сортувати процеси за різними параметрами:

| Ключ / Поле | Опис                                     |
| ----------- | ---------------------------------------- |
| P         | Сортування за використанням CPU (%)        |
| M         | Сортування за використанням пам’яті (%)    |
| T         | Сортування за часом роботи процесу (TIME+) |
| N         | Сортування за PID (ідентифікатор процесу)  |

Перемикатись можна задопомогою комбінації клавіш "Shift + P/M/T/N)

2.4. **Які команди для завершення роботи процесів ви знаєте?  

| Команда        | Опис                                                            | 
| -------------- | --------------------------------------------------------------- | 
| `kill`         | Надсилає сигнал процесу за PID                                  | 
| `killall`      | Завершує всі процеси із заданою назвою                          | 
| `pkill`        | Завершує процеси за іменем або шаблоном                         | 
| `xkill`        | У графічному середовищі завершує процес, на який клацнули мишею | 
| `top` / `htop` | Можна вибрати процес і натиснути `k` (kill)                     |

- Які дії можна зробити в терміналі за допомогою комбінацій Ctrl + C, Ctrl + D та Ctrl + Z?

Комбінація Дія

    Ctrl + C Зупиняє (перериває) виконання поточного процесу (посилає сигнал SIGINT).
    Ctrl + D Завершує введення (EOF) або виходить із терміналу (якщо натиснуто в порожньому рядку).
    Ctrl + Z Призупиняє виконання процесу (посилає сигнал SIGTSTP) і переводить його у фоновий режим.


- *Чим відрізняється фоновий процес від звичайного. Де вони використовуються?

    Звичайний (foreground) Виконується у терміналі, блокує введення, поки не завершиться.
    Фоновий (background) Виконується у фоновому режимі, не блокує термінал, дозволяє вводити інші команди.


Запуск у фоновому режимі: bg

- *Опишіть наступні команди та поясніть що вони виконують – команда jobs, bg, fg.

Команди:
    
    jobs - Показує список активних задач у поточному сеансі терміналу (фон/призупинені).
    bg - Переводить призупинену задачу у фоновий режим (продовжує виконання).

1. Яке призначення директорії /proc в системах Linux. Яку інформацію вона зберігає?  
 Директорія /proc — це віртуальна файловa система, що надає інформацію про ядро, процеси та апаратні ресурси у режимі реального часу.  
Її вміст не зберігається на диску — він формується динамічно ядром під час звернення.  
Основне призначення:  
- Дозволяє користувачам і програмам отримувати системну інформацію безпосередньо з ядра.  
- Служить для керування параметрами ядра (через /proc/sys).  
- Може бути використана для моніторингу процесів, пам’яті, CPU, пристроїв, файлових систем тощо.  
2. Як серед будь-яких трьох процесів динамічно визначати, який з них в поточний момент часу використовує найбільший обсяг пам'яті? Який відсоток пам’яті він споживає від загального обсягу?

- -p [1,2,3] - PID - Номер процесу  
- -o — які поля показати (pid, comm — назва процесу, %mem — відсоток пам’яті) 
- --sort=-%mem — сортування за спаданням використання пам’яті.

5. *Які додаткові можливості реалізує htop в порівнянні з top?

`htop` — це покращена версія утиліти `top`, що надає розширені можливості моніторингу процесів у Linux.

Переваги `htop` над `top`:

    Має інтерактивний графічний інтерфейс з кольоровим оформленням.
    Підтримує навігацію клавішами зі стрілками, що спрощує вибір і керування процесами.
    Дає змогу завершувати або змінювати пріоритет процесів (kill, renice)** без введення PID вручну.
    Відображає зручні графіки завантаження CPU, пам’яті, swap.
    Можна сортувати процеси за будь-яким полем (CPU, пам’ять, час тощо).
    Показує повну ієрархію процесів (tree view).
