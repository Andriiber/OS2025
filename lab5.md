2.2. *Розкрийте поняття FHS. Як даний стандарт використовується в контексті файлових систем? 

 FHS (Filesystem Hierarchy Standard) — це стандарт організації файлової системи Linux, який визначає, які каталоги повинні існувати в системі та яке їх призначення.

 Основна ідея:

    FHS створює єдину логічну структуру для всіх дистрибутивів Linux, щоб:
    
    користувачі та адміністратори могли орієнтуватися незалежно від дистрибутиву,
    
    програми могли знаходити потрібні файли в стандартних місцях.

5. Опишіть дії, які виконують команди для переміщення по системі каталогів:

- ```команда cd /``` - Переходить до кореневого каталогу файлової системи.

- ```команда cd /home``` - Переходить до каталогу /home (Зазвичай там знаходяться домашні теки користувачів, наприклад /home/user)

- ```команда cd ~```	- Переходить до домашньої теки поточного користувача.

- ```команда cd (без аргумента)``` - Робить те саме, що й cd ~. Переходить у домашню теку поточного користувача.

- ```команда cd ..```	- Переходить на один рівень вище у структурі тек.

- ```команда cd ../..```	- Переходить на два рівні вище у структурі тек.

- ```команда cd -``` - Повертає у попередню теку, де ти був до цього.

3. *Яким чином в терміналі можна додати інформацію в порожній файл?

       Якщо файл уже створений і порожній — можна додати текст так:
       
       echo "Тут якийсь текст" > file.txt - (заміщує вміст файлу)
       
       Або, щоб додати до існуючого вмісту, не стираючи його:
       
       echo "Новий рядок тексту" >> file.txt


4. **Як скопіювати та видалити існуючий каталог? Чи буде відмінність в командах, якщо каталог буде не порожній при цьому
        
  Копіювання:
        
    Якщо каталог порожній:
         
    cp -r source_dir destination_dir
         
    Ключ -r (recursive) потрібен, навіть якщо всередині є файли.

  Видалення:

    Якщо каталог порожній:
    
    rmdir myfolder
    
    
    Якщо каталог не порожній:
    
    rm -r myfolder
    
    Ключ -r дозволяє видаляти рекурсивно весь вміст.


5. **У якому з наведених нижче прикладів відбувається переміщення файлу? його перейменування? одночасно обидві дії?

       mv /work/tech/comp.png. /Desktop --- Файл переноситься в іншу директорію, назва не змінюється

       mv /work/tech/comp.png. /Desktop/computer.png --- Файл переноситься в інше місце і отримує нову назву
