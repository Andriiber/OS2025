1. *Охарактеризуйте поняття скриптового сценарію у командній оболонці.

       Скриптовий сценарій (скрипт) — це текстовий файл, який містить послідовність команд для виконання в командній оболонці (наприклад, Bash у Linux). Скрипти дозволяють автоматизувати повторювані завдання, обробляти дані, запускати програми та управляти системними ресурсами без ручного введення кожної команди.

2. *Яким чином створюються та редагуються скрипти, що треба зробити щоб запустити скрипт?

       Створення та редагування: скрипти створюються в текстових редакторах (наприклад, vi, nano) та зберігаються з будь-яким ім’ям, часто з розширенням .sh.
       
       Приклад створення:
       vi myscript.sh
       
       Зміст скрипта:
       #!/bin/bash
       echo "Hello, world!"
       
         Запуск скрипта:
       Зробити файл виконуваним:
       chmod a+x myscript.sh
       
       Виконати його:
       ./myscript.sh
       
       Або через інтерпретатор:
       bash myscript.sh

3. **Які основні компоненти материнської плати ви знаєте?

       Процесорний роз’єм (CPU socket) — місце для встановлення процесора.
       
       Слоти оперативної пам’яті (RAM slots) — для модулів пам’яті.
       
       Чіпсет — координує взаємодію між CPU, пам’яттю та периферією.
       
       Слоти розширення (PCI, PCIe) — для додаткових карт (відео, звук, мережа).
       
       Порти введення/виведення — USB, Ethernet, аудіо, відео.
       
       BIOS/UEFI — прошивка для старту системи.
       
       Конектори живлення — підключення живлення від блока живлення.
       
       Слоти для накопичувачів — SATA, M.2.

4. **Коротко охарактеризуйте для яких пристроїв оперують поняттями MBR та GPT?

       MBR (Master Boot Record) та GPT (GUID Partition Table) — це схеми розділів жорстких дисків та SSD.
       
       MBR: старіша схема, підтримує до 2 ТБ диска та максимум 4 первинні розділи.
       
       GPT: сучасніша, підтримує дуже великі диски (>2 ТБ) і до 128 розділів, використовується у системах з UEFI.
       
       Використовуються для розділення жорстких дисків і SSD на логічні області для даних та системи.

5. **В чому суть операції монтування, для чого вона потрібна?

       Монтування (mounting) — це процес підключення файлової системи диска або іншого пристрою до директорії у файловій системі Linux. Монтування буквально включає данні з ночія до одної з вже існуючих дерикторій.
       
       Після монтування можна отримувати доступ до файлів і каталогів на диску як до звичайних файлів системи.
       
       Без монтування операційна система не бачить вміст носія.
